### Re-research for: Egg portability across Pterodactyl-fork ecosystem
Status: RESOLVED

The original cited repo `pelican-eggs/eggs` was indeed archived in May 2024 and reorganized into category-specific repos. As of 2026 the live community-maintained Pelican egg repos under the `pelican-eggs` GitHub organization are: `minecraft`, `games-steamcmd`, `games-standalone`, `generic`, `voice`, `chatbots`, `database`, `software`, `monitoring`, `storage`, and `yolks` (Docker images) [src:https://github.com/pelican-eggs | authority:GitHub org listing]. The org explicitly describes itself as "a community-maintained collection of 'eggs' for Pelican... configurations for the Pterodactyl panel" â€” framing the eggs as a shared asset across both panels.

The portability claim survives intact at the JSON-shape level. Each Pelican egg directory now ships two files side-by-side: a native `egg-<name>.yaml` (Pelican's newer format) and a `pterodactyl-egg-<name>.json` exported in the Pterodactyl schema. Comparing the canonical Pterodactyl egg-paper.json (`meta.version: "PTDL_v2"`, `_comment: "DO NOT EDIT: FILE GENERATED AUTOMATICALLY BY PTERODACTYL PANEL - PTERODACTYL.IO"`) [src:https://raw.githubusercontent.com/pterodactyl/panel/develop/database/Seeders/eggs/minecraft/egg-paper.json | authority:Pterodactyl canonical] against the Pelican-hosted equivalent (`meta.version: "PTDL_v2"`, identical `_comment`, `update_url` pointing back to the pelican-eggs raw URL, exported 2025-12-31) [src:https://raw.githubusercontent.com/pelican-eggs/minecraft/main/java/paper/pterodactyl-egg-paper.json | authority:pelican-eggs raw file] confirms byte-level compatibility of the JSON shape â€” same schema version, same generator comment string, same field layout. Pelican is exporting in Pterodactyl's own format on purpose, so a Pterodactyl panel can consume the file unchanged.

Pelican's official FAQ confirms the migration path is supported in the other direction too: asked whether existing Pterodactyl eggs work on Pelican, the answer is "Yes! We have a newer version of them planned, but the original ones will **always** work" [src:https://pelican.dev/faq | authority:Pelican project FAQ]. The "newer version" aligns with the YAML format now appearing in the category repos, but the JSON PTDL_v2 form remains a first-class, dual-published artifact rather than a deprecated legacy. So the original claim â€” that the egg JSON shape is preserved and the migration path is intact across the Pterodactyl-fork ecosystem â€” holds; the only correction to the original citation is that the `pelican-eggs/eggs` monorepo became the per-category `pelican-eggs/*` repos (e.g., `pelican-eggs/minecraft` [src:https://github.com/pelican-eggs/minecraft | authority:GitHub repo listing]).
