# Re-research: pyrodactyl-claims

### Re-research for: Pyrodactyl performance claims
Status: STILL-WEAK

**Verified the original quote, but not the underlying numbers.** The README at [github.com/pyrohost/pyrodactyl](https://github.com/pyrohost/pyrodactyl) does indeed contain the claims as cited: "Pyrodactyl's initial download size is over **170 times smaller than leading Pterodactyl forks, including Pelican**" and "load times are, on average, **over 16 times faster** than other closed-source Pterodactyl forks and Pelican" [src:https://github.com/pyrohost/pyrodactyl | authority:vendor primary]. Both numbers are linked only to imgur screenshots (i.imgur.com/tKWLHhR.png and i.imgur.com/28XxmMi.png) with no description of methodology, Pelican version, hardware, network conditions, or measurement tooling. The Pyrodactyl docs site at [pyrodactyl.dev/docs/pyrodactyl](https://pyrodactyl.dev/docs/pyrodactyl) similarly states the numbers without any benchmark scripts, version specs, or test conditions [src:https://pyrodactyl.dev/docs/pyrodactyl | authority:vendor primary].

**No independent verification exists.** Targeted searches for "pyrodactyl benchmark", "pyrodactyl review", and "pyrodactyl vs pelican" return only Pyrodactyl's own properties (github, pyrodactyl.dev, deepwiki mirror, template-listing pages like builtatlightspeed.com and tailwindresources.com) plus unrelated namesakes (Pyrodactyl Games, an indie game studio at pyrodactyl.com). No third-party blog post, hosting-provider write-up, academic comparison, or reproducible benchmark was located. Community discussions on Admincraft and LowEndSpirit ([lowendspirit.com/discussion/9906](https://lowendspirit.com/discussion/9906/any-good-alternatives-to-pterodactyl)) discuss Pterodactyl-vs-Pelican feature/stability but make no measurements on Pyrodactyl's size or speed claims [src:https://lowendspirit.com/discussion/9906/any-good-alternatives-to-pterodactyl | authority:community forum]. Pelican's own comparison page ([pelican.dev/docs/comparison](https://pelican.dev/docs/comparison/)) doesn't engage with Pyrodactyl at all — it only compares against vanilla Pterodactyl [src:https://pelican.dev/docs/comparison/ | authority:competitor primary]. Notably, the linked "pyro.host" lands on a game-server hosting product page that doesn't mention Pyrodactyl, Pterodactyl, or Pelican by name — so there is no second corroborating Pyro/Pyrohost surface for the claim either [src:https://pyro.host | authority:vendor primary, tangential].

**"Safer" and "more accessible" are partially defined but not benchmarked.** The README does give concrete (if thin) definitions: "safer" = "modern architecture means most severe and easily exploitable CVEs simply do not exist" plus "SRI and integrity checks for production builds" — no specific CVEs are enumerated and no comparative CVE counts are given. "More accessible" = "entirely keyboard-navigable, even context menus, and screen-readers are easily compatible," with a generic reference to "latest Web accessibility guidelines" but no named WCAG level or audit [src:https://github.com/pyrohost/pyrodactyl | authority:vendor primary]. These are real, testable feature claims, but no independent accessibility audit (axe report, WCAG conformance statement, a11y blog review) was found.

**Guidance for the synthesizer:** Keep the claim as **marketing-attributed**. Frame as "Pyrodactyl self-reports a ~170x smaller initial download and ~16x faster load times vs. Pelican, with the comparison evidence consisting only of two screenshots in the project README; no third-party benchmark corroborates these figures." For "safer" and "more accessible," replace the marketing words with the concrete features the README actually names: SRI/integrity checks for production builds, and full keyboard navigation plus screen-reader compatibility — and note that neither has been independently audited. Do NOT assert the 170x/16x numbers as fact.
